CSS_3

$1 盒子模型

$1.1 网页布局过程

1. 准备相关网页元素，网页元素基本都是盒子Box
2. 利用CSS设置好盒子样式，摆放好位置
3. 往盒子里装内容

$1.2 盒子模型组成

外边距margin，边框border，内边距padding，内容content

1.2.1 边框border

  • 语法：border: border-width | border-style | border-color
    • width：像素值
    • style：常用实线solid，虚线dashed，点线dotted
    • color：同样是三种写法
  • 简写时候没有顺序
  • 可以指定边框border-top | bottom | left | right，符合层叠性原理，因此可以：
border: 1px solid blue;
border-top: 1px solid red;
  • border-collapse：设置为collapse时会合并表格相邻边框
  • 边框会影响盒子的实际大小，也就是说width:200px border: 10px的实际width为220

1.2.2 内边距padding

  • 语法：padding-top | bottom | left | right: 像素值

  • 默认四个内边距都为0px

  • 可以有1~4个值

    • 一个值，表示上下左右
    • 两个值，表示上下，左右
    • 三个值，表示上，左右，下
    • 四个值，表示上，右，下，左

  • 内边距会影响盒子的实际大小，具体性质同边框

  • 如果盒子本身没有width或height属性，则padding不会撑开盒子大小

1.2.3 外边距margin

  • 语法：margin-top | bottom | left | right: 像素值
  • 默认四个外边距都为0px
  • 可以有1~4个值，具体同内边距padding
  • 应用：
    • 块级元素水平居中： margin: 0 auto;
    • 行内元素或行内块元素水平居中：父元素添加text-align: center;
  • 外边距合并
    • 相邻块级元素：最终的margin为第一个元素的margin-bottom和第二个元素的margin-top中最大值。所以尽量只给一个盒子添加margin
    • 嵌套级块元素：父元素margin-top为20px，子元素margin-top为10px，最终子元素不会距离父元素10px，反而父元素有了20px的上外边距
      • 为父元素定义上边框
      • 为父元素定义上内边距
      • 为父元素添加overflow: hidden;，溢出隐藏

1.2.4 清除网页元素默认内外边距

* {
	padding: 0px;
	border: 0px;
}

$1.3 圆角边框

  • 语法：border-radius: 2px;

    • 原理：圆与边框的相切部分，即1/4圆

  • 圆形盒子做法：border-radius: 正方形边长的一半或者border-radius: 50%;

  • 圆角矩形：border-radius: 矩形高度的一半

  • 可以有1~4个值，分别表示左上，右上，右下，左下，即border-top-left-radius等，

    • 一个值，表示四个方位
    • 两个值，表示左上右下，左下右上
    • 三个值，表示左上，左下右上，右下
    • 四个值，表示左上，右上，右下，左下

$1.4 盒子阴影

  • 语法：box-shadow: h-shadow v-shadow blur speard color inset
    • h-shadow：必需，水平阴影的位置，可以为负值，即阴影的左右移动
    • v-shadow：必需，垂直阴影的位置，可以为负值，即阴影的上下移动
    • blur：模糊距离，即影子虚实，0为纯黑色
    • spread：阴影尺寸，即影子大小
    • color：阴影颜色
    • inset或outset：内部或外部阴影，不写默认outset，写上outset会出错，在设置为内部阴影时写上
  • 盒子阴影不改变盒子实际大小
  • 可以结合hover使用，加入过渡动画会更好

$1.5 文字阴影

  • 语法：text-shadow: h-shadow v-shadow blur color
    • 具体意思见盒子阴影

$2 浮动

$2.1 传统网页布局三种格式

  • 普通流(标准流)：标签按照规定好的默认方式排列，是最基本的布局方式
    • 块级元素独占一行
    • 行内元素从左到右顺序排列
  • 浮动，可以改变标签默认排列方式，完成许多标准流无法完成的布局效果。最常用的是让块级元素一行显示
    • 多个盒子纵向排列找标准流
    • 多个盒子横向排列找浮动
  • 定位

一个实际页面中基本都包含了这三种布局方式

$2.2 浮动🔴

$2.2.1 基础

语法：float: none | left | right

​ 用于创建浮动框，将其移动到一边，直到左边缘或右边缘触及包含块或另一个浮动框的边缘

$2.2.2 特性

  • 脱离标准流的控制(浮)，移动到指定位置(动)，俗称脱标
  • 浮动的盒子不再保留原先的位置
  • 如果多个盒子设置了浮动，则它们会按照属性值一行内显示并顶端对齐，如果父级宽度装不下，则会另起一行对齐
  • 浮动元素具有行内块元素特性
    • 如果块级元素没有设置宽度，默认宽度和父级一样宽，但是添加浮动后，它的大小由内容来定
    • 浮动的盒子中间没有缝隙
  • 浮动元素经常和标准流父级搭配
    • 用标准流的父级元素排列上下位置
    • 用浮动排列左右位置

$2.3 常见网页布局

$2.4 浮动总结

  • 浮动元素经常和标准流父级搭配
  • 一个盒子里面多个盒子，如果一个盒子浮动了，那么其它兄弟也应浮动
    • 浮动的盒子指挥影响浮动盒子后面的标准流，不会影响前面的标准流

$2.5 清除浮动

​ 由于父级盒子很多情况下，不方便给高度，但是子盒子浮动又不占有位置，最终使得父盒子高度会变为0，影响下面的标准流盒子

$2.5.1 基础

  • 本质：清除浮动元素脱离标准流造成的影响
  • 前提：父盒子本身没有高度
  • 语法：clear: left | right | both;，实际开发中只使用both

$2.5.2 清除浮动方法

闭合浮动，只让浮动在父盒子内部影响，不影响之外的其他盒子

  1. 额外标签法，也称隔墙法
    • 添加空块级元素标签：<div style="clear: both"></div>
    • 优点：通俗易懂，书写方便
    • 缺点：添加无意义标签，结构化较差
  2. 父级添加overflow属性
    • overflow: hidden | auto | scroll
    • 优点：代码简洁
    • 缺点：无法显示溢出文本
  3. 父元素添加after伪元素
    • 缺点：照顾低版本浏览器
.clearfix:after {
    content: "";
    display: block;
    height: 0;
    clear: both;
    visibility: hidden;
}
.clearfix {
    *zoom : 1;
}
/*然后父元素加上class="clearfix"即可*/

  1. 父元素添加双伪元素

    • 优点：代码相比简洁，小米，腾讯均采取此做法

    • 缺点：照顾低版本浏览器

.clearfix:before,
.clearfix:after {
    content: "";
    display: table;
}
.clearfix:after{
    clear: both;
}
.clearfix {
    *zoom: 1;
}
/*然后父元素加上class="clearfix"即可*/
